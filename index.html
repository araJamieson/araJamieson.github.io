Caveats:

  • I am not sure whether the definitions of the borough boundaries used in this data are necessarily the same as those used when defining the deaneries.
  • Since the data was drawn up in 2018, it would seem reasonable to assume that data for 2011 would coincide with the census figures for 2011, available from the same website. In fact there appear to be discrepancies of at least 10% between the two in some cases, and there is no obvious explanation for this. This may cast some doubt on the reliability of the data.

Overview Tool version: $softwareVersion$;   Data dated: $dataDate$. Data is more than $warnAfterDays$ days old. This works in much the same way as Google maps (ie you can scroll, zoom, etc). The program shows a map of the UK, centred on the area covered by the Hackney, Islington and Tower Hamlets deaneries. Churches are marked with pegs — PINK for churches in the Hackney deanery, PURPLE for Islington, and GREEN for Tower Hamlets. Each carries an icon to indicate the type of church — a candle for high church, a book for evangelical, and a handshake for liberal (and a question mark where this is still to be established). Selecting churches Click on a church to select it. Click it again to deslect it. You can select more than one church at once if you wish. Where known, the parish areas of selected churches are highlighted. The areas are colour coded according to GC (green through red, with green corresponding to 1, or grey where unknown). When enabled, a pop-up appears when you select a church, giving information about the church. Displaying numerous pop-ups may clutter the screen, so there is a checkbox on the menu which lets you control whether or not they are shown. Items in blue on the popup are hyperlinks to pages where more detailed information is available. The information available differs between the various churches, according to what I was able to find. The popup for each church includes a button which lets you limit the display to showing only churches within a given distance of the one you have selected. So long as one or more churches are selected, aggregate information about the churches will appear at the bottom of the page. It may sometimes be the case that the aggregate information is not available for all of the churches selected. It is made clear where this is the case. Tracking EXPERIMENTAL ONLY. Do not rely upon this feature until you have built up some measure of confidence that it is working on your device. As an aid to navigation, the program has an experimental feature which lets you track your position as you move around. This is turned off by default. You control it using a checkbox on the menu. When you turn the facility on, you will be asked for permission to access location data. Note that even if you give permission, the program will have access to such data only if you have permitted it in your device’s settings and also (where relevant) in your browser. While the feature is active, it will redraw the map every 5-10 seconds (possibly also changing the zoom settings) so that it is centred on your present location. This constant redrawing means that you cannot really browse the map while tracking is turned on — any manual scrolling you do will be undone every few seconds. In view of this, I recommend turning the feature off any time you want to browse the map, any time you are using it as the basis for analysis or discussion, or any time you are working on a non-mobile device. About the data The information in the tool has come from a variety of sources. The basic framwork has been derived from the London Diocesan Directory. I have supplemented this with information from Wikipedia, with details taken from the various church’s own websites, and with material derived from a number of other miscellaneous sources. The information from the Diocesan Directory should in theory be the most reliable — although with the caveat that I have had to try to extract this information automatically, and taking information from web pages in this way is far from straightforward, and relies upon different pages being put together in a consistent manner. In general the information from the other websites is less important, since most of it merely provides links to other places, and it will be readily apparent if they are wrong. Having said that, the parish population information comes from Wikipedia, and I have no way of verifying that, nor of determining if it is up to date. Note that the data from the Diocesan Directory covers more than just churches (it includes various chaplaincies, for instance). I have removed this information from the data presented here, but if you feel that some of it should be included (or that some of the data which the tool actually presents should not be appearing) the list is easily amended — support details below. About the tool The tool comprises an Excel spreadsheet in which I have gathered the various pieces of information, and then a collection of webby things, one of them derived from the spreadsheet, and the rest hand-crafted. It is possible that the spreadsheet may be useful in its own right. The mapping data is all open source (ie it’s not Google Maps data), taken from OpenStreetMap. The software which turns this data into displayable form is also open source (leaflet.js). I also rely upon software from mapbox. This is not public domain, but they do make freely available a public key for limited general use, and this seems to be adequate.
